## Plotting Blocks

Plotting Blocks is a small **HTML/CSS** layout exercise focused on practicing structural markup and box layout.

### Overview

- **Goal**: Arrange multiple colored or content “blocks” on a page using semantic HTML and clean CSS.
- **Core idea**: Build a grid or series of panels that align consistently and respond visually to spacing, borders, and typography.
- **Tech stack**: HTML5, CSS3 (no JavaScript required for the core assignment).

### Project Structure

The main project files live in the nested `PlottingBlocks/` folder:

- `index.html` – Markup for all of the blocks/sections.
- `style.css` – Styles for sizing, spacing, alignment, and colors.

### How to Run Locally

1. Open `PlottingBlocks/index.html` directly in your browser, or
2. Serve the folder via a simple HTTP server (for example `python -m http.server` from this directory).

### How to Use / Explore

- Resize the browser window to see how the layout behaves at different widths.
- Inspect elements to see how margins, padding, and display modes (e.g., `block`, `inline-block`, `flex`) are applied.

### Design Notes

- Emphasizes **visual balance** and consistency:
  - Even spacing between blocks.
  - Consistent border radii and colors.
  - Clear typographic hierarchy, if text is present.
- Serves as a **pure layout** piece in a portfolio, showing attention to spacing and structure.

### Portfolio Highlights

- Demonstrates:
  - Comfort with CSS box model and layout.
  - Clean separation between content (`index.html`) and presentation (`style.css`).
  - Ability to translate a static wireframe or sketch into well‑structured markup.
